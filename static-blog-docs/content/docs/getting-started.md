# Getting Started

Welcome to the documentation for the **Static Blog & Documentation Site**. This project is built using **Next.js**, **TypeScript**, **Tailwind CSS**, and **Markdown**. It demonstrates how to create a static blog using Static Site Generation (SSG) while also building a simple documentation website with nested layouts and a persistent sidebar.

The application is designed to keep content separate from the source code. Instead of storing blog posts and documentation in a database, all content is written in Markdown files located inside the `content` directory. During the build process, Next.js reads these files, converts them into HTML, and generates static pages that can be served quickly to users.

This documentation section is organized into multiple pages, each covering a specific topic. A sidebar is available throughout the documentation to help users navigate between sections without leaving the documentation layout. This structure makes the content easier to explore and provides a better reading experience.

Whether you are reviewing this project for learning purposes or as part of an assignment, the documentation will guide you through the project setup, installation process, and folder organization. Each section is written to be simple, clear, and easy to follow.

## Project Goals

The primary objectives of this project are:

- Build a static blog using Markdown files.
- Generate pages using Static Site Generation (SSG).
- Create a documentation website using nested layouts.
- Organize documentation with a persistent sidebar.
- Maintain a clean and scalable project structure.

## Conclusion

This project demonstrates how modern Next.js features can be combined with Markdown to create a lightweight, fast, and easy-to-maintain website. By separating content from application logic, developers can focus on writing content while keeping the codebase clean and organized.
